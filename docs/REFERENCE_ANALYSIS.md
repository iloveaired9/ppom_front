## 📊 Executive Summary

뽐뿌 사이트의 **현재 상태**는 CSS 변수 기반의 기본 다크모드 구조를 가지고 있지만, **네이버의 고급 다크모드 구현 방식**과는 상당한 차이가 있습니다.

| 항목 | 뽐뿌 (현재) | 네이버 (목표) | 평가 |
|------|-----------|------------|------|
| **CSS 변수 사용** | ✅ 있음 (20개) | ✅ 있음 | 둘 다 사용 |
| **다크모드 클래스** | ❌ 없음 | ✅ `dark-mode` | 구현 필요 |
| **Color-scheme Meta** | ❌ 없음 | ✅ 있음 | 추가 필요 |
| **배경색 대비** | ⚠️ 약함 | ✅ 높음 | 개선 필요 |
| **색상 일관성** | ⚠️ 제한적 | ✅ 완벽 | 강화 필요 |
| **접근성 (WCAG)** | ⚠️ 미검증 | ✅ AA 충족 | 검증 필요 |

## 🔍 상세 분석

### 1. 뽐뿌의 현재 다크모드 색상 체계

#### 발견된 CSS 변수 (20개)
```
기본 색상:
  --color-m-basic: #fff                     (기본 배경)
  --color-m-dark-base: #000                 (검정)
  --color-m-text-basic: #333                (텍스트)

명도 관련:
  --color-m-light-gray: #e3e3e3             (밝은 회색)
  --color-m-gray-base: #ccc                 (회색)
  --color-dark-bottom-line: #2f2f2f         (어두운 분할선)

강조 색상:
  --color-m-blue4: #0068ff                  (파란색)
  --color-m-blue3: #0267cf                  (짙은 파란색)
  --color-m-blue: #647aff                   (연한 파란색)
  --color-m-dark-blue: #5482d0              (다크 파란색)
  --color-m-red: #ff4800                    (주황-빨강)

접근성:
  --color-m-dark-base-text-sub: #8b8b8b     (부텍스트)
  --color-m-visited: #858585                (방문한 링크)
```

#### 문제점
❌ **색상 이름이 모호함** (예: `--color-m-blue` 등 순서가 일관성 없음)
❌ **다크모드 전용 변수 부족** (대부분 라이트모드 기준)
❌ **명도 계층화 없음** (background, secondary, tertiary 구분 없음)
❌ **강조색 선택이 명확하지 않음** (파란색 또는 주황색?)

### 4. HTML/CSS 구조 비교

#### 뽐뿌 현재 구조
```html
<!-- HTML -->
<html lang="ko" class="">  <!-- 다크모드 클래스 없음 -->
  <head>
    <!-- color-scheme meta 없음 -->
    <meta name="theme-color" content="...">
  </head>
  <body>
    <!-- 콘텐츠 -->
  </body>
</html>

/* CSS - 5개 스타일시트 */
/* 대부분 인라인 스타일 또는 클래스 기반 -->
```

#### 네이버 스타일 구조
```html
<!-- HTML -->
<html lang="ko" class="dark-mode">  <!-- 다크모드 클래스 있음 -->
  <head>
    <meta name="color-scheme" content="light dark">  <!-- 추가됨 -->
    <meta name="theme-color" content="...">
  </head>
  <body>
    <!-- 콘텐츠 -->
  </body>
</html>

/* CSS - CSS 변수 기반 */
:root {
  --nv-bg-primary: #111111;
  --nv-text-primary: #FFFFFF;
  /* ... */
}

@media (prefers-color-scheme: dark) {
  :root { /* 다크모드 변수 */ }
}

html.dark-mode {
  /* 명시적 다크모드 클래스 */
}
```

### 5. 주요 차이점 분석

#### ✅ 뽐뿌 현재 장점
1. **CSS 변수 기반** - 유지보수 용이
2. **기본 다크모드 고려** - 초기 구조 있음
3. **다양한 색상 옵션** - 파란색/주황색 선택 가능

#### ❌ 뽐뿌 현재 단점
1. **시스템 설정 미감지** (`prefers-color-scheme` 없음)
2. **명확한 다크모드 클래스 없음** (`.dark-mode` 클래스 미사용)
3. **명도 계층화 부족** (배경색, 텍스트색 세분화 없음)
4. **색상 명명 규칙 불명확** (순서 체계 없음)
5. **강조색 혼란** (파란색이 주인지, 주황색이 주인지 불명확)
6. **접근성 검증 없음** (WCAG 대비 기준 미충족 가능성)

### 6. 마이그레이션 권장사항

#### 🎯 Phase 1: 색상 시스템 재정리

**현재 상태:**
```css
/* 뽐뿌 (혼란스러움) */
--color-m-blue: #647aff;
--color-m-blue3: #0267cf;
--color-m-blue4: #0068ff;
--color-m-dark-blue: #5482d0;
--color-m-red: #ff4800;
```

**목표 상태:**
```css
/* 네이버 (명확함) */
:root {
  --nv-bg-primary: #FFFFFF;
  --nv-bg-secondary: #F5F5F5;
  --nv-text-primary: #000000;
  --nv-accent: #00C73C;
  --nv-link: #0066CC;
}

@media (prefers-color-scheme: dark) {
  :root {
    --nv-bg-primary: #111111;
    --nv-bg-secondary: #1A1A1A;
    --nv-text-primary: #FFFFFF;
    --nv-accent: #00C73C;
    --nv-link: #00D5FF;
  }
}
```

#### 🎯 Phase 2: HTML 업그레이드

```html
<!-- 추가할 항목 -->
<html lang="ko" class="dark-mode">
  <head>
    <!-- 추가: color-scheme 메타 -->
    <meta name="color-scheme" content="light dark">
  </head>
</html>
```

#### 🎯 Phase 3: 색상 강조 결정

**선택이 필요한 부분:**

| 항목 | 옵션 | 추천 |
|------|------|------|
| **주 강조색** | 파란색 (#0068FF) or 초록색 (#00C73C) | ✅ 초록색 (네이버) |
| **부 강조색** | 주황색 (#FF4800) or 시안색 (#00D5FF) | ✅ 시안색 (네이버) |
| **에러색** | 빨강 | ✅ 유지 |
| **성공색** | 초록색 | ✅ 주강조색과 통일 |
